Trust and Safety

Security Scanning on PULL.md

Every markdown asset is scanned before publish and on moderator edits. We expose clear scan status on cards and keep technical details available for users who want deeper review.

Marketing Version Technical Version

Marketing Version

Simple explanation for buyers and creators.

PULL.md is designed for safe, reusable markdown commerce. Each listing is scanned for risky patterns before it can be published, and scanned again when content is edited by moderators.

  • Checks run on every publish and moderation edit
  • Clear status on cards: clean, warning, or blocked
  • High-confidence risks can be blocked in enforce mode
  • No scanner can guarantee safety, so review content before use

Technical Version

Scanner modules, checks, and package inventory.

Scanner Modules

  • unicode: hidden Unicode controls (zero-width, bidi override, invisible operators, tag chars)
  • confusable: lookalike or homoglyph token detection
  • markdown_structure: dangerous HTML tags, inline handlers, hidden structures and comments
  • url_safety: dangerous schemes, malformed links, internal targets, domain and IP risk checks
  • prompt_injection: high-risk instruction phrase detection
  • secret_leak: API key and token pattern detection

Optional Reputation Layer

  • url_reputation via URLhaus host lookup (env-gated, cached, timeout-bound)

Outcomes

  • clean: no findings
  • warn: non-blocking findings
  • block: critical findings, blocked when enforce mode is enabled
